# Contributing to ever2e-rom

Thanks for your interest in improving ever2e-rom.

## Development Setup

1. Review architecture and memory maps:
   - `README.md`
2. Keep changes scoped and testable:
   - include a short validation note in your PR

## Coding Guidelines

- Keep changes focused and small.
- Preserve Apple IIe compatibility requirements documented in `README.md`.
- Add or update tests/check scripts with behavior changes.
- Keep docs and implementation details in sync.

## Pull Requests

- Use clear commit messages.
- Describe what changed and why.
- Include test evidence (what you ran and observed).
- If behavior changes, update `README.md` and related docs.

## Before Opening a PR

- Local validation for changed components is complete.
- Any new scripts/build steps are documented.
- Memory map, vector, and soft-switch assumptions remain consistent.
